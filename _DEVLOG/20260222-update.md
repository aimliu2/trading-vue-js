### Recommended Debounce Times by Scenario
- **Search Input/Autocomplete**: 250ms–300ms is the standard for feeling instant without overloading the server.
- **Fast-Paced UI** (Button Click/Toggle): 50ms–100ms. If the action is critical (e.g., in a game), this provides immediate feedback while filtering physical switch "bounces".
- **Form Validation**: 500ms. This gives the user enough time to type a few characters (e.g., an email address) before showing an error, preventing premature "invalid" warnings.
- **Saving Data/Drafts**: 1000ms+ (1 second). This protects against spamming API calls on slow networks without sacrificing the user's progress
