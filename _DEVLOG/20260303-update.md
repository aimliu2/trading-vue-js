# Fix Vue 3 Runtime Errors after Webpack → Vite Migration

## Problem
After switching `src/main.js` from Vue 2 (`new Vue(...)`) to Vue 3 (`createApp(...)`), the chart rendered a blank canvas. No overlays registered their drawing layers.

## Bugs Fixed

### 1. [overlay.js](../src/mixins/overlay.js) — Blank canvas (root cause)
Vue 2 hack in `mounted()` tried to override `this.$emit` by checking if it was a plain function via `new Function('return ' + this.$emit)()`.
In Vue 3, `this.$emit` is `emit.bind(null, instance)` — a bound native function — so `.toString()` returns `"function () { [native code] }"`.
Wrapping that in `new Function(...)` throws `SyntaxError: Unexpected identifier 'code'`.
The `catch(e) { return }` block caught it silently, exiting `mounted()` before any `new-grid-layer` event was emitted → blank canvas.

**Fix:**
- Removed the `new Function(...)` try/catch block entirely
- Replaced all `this._$emit(...)` calls with `this.$emit(...)`
- Renamed `beforeDestroy()` → `beforeUnmount()` (Vue 3 lifecycle rename)

### 2. [UxLayer.vue](../src/components/UxLayer.vue) — `this.$emit` in `<script setup>`
`const on_custom_event = (event) => this.$emit('custom-event', event)` — `this` is `undefined` in `<script setup>`, throws TypeError at runtime.

**Fix:** Added `const emit = defineEmits(['custom-event'])` and replaced with `emit('custom-event', event)`.

### 3. [Chart.vue](../src/components/Chart.vue) — Missing `defineExpose`
`<script setup>` components are closed by default. TradingVue.vue accesses `chartRef.value.*` for many internal state values and methods, all of which returned `undefined`.

**Fix:** Added `defineExpose` exposing:
- `ti_map`, `ohlcv`, `range`, `cursor`, `activated`, `interval`, `interval_ms`
- `goto` (aliased from `jumpToTime`), `setRange`

### 4. [Sidebar.vue](../src/components/Sidebar.vue) — Missing `defineExpose`
Section.vue calls `sidebarRef.value.renderer.rezoom_range(...)` — `renderer` was not accessible.

**Fix:** Added `defineExpose({ renderer: sidebarPixel })`.

### 5. [Keyboard.vue](../src/components/Keyboard.vue) — Missing `defineExpose`
Chart.vue calls `keyboardRef.value.register(event)` and `.remove(event)` — neither was accessible.

**Fix:** Added `defineExpose({ register, remove })`.

### 6. [App.vue](../src/App.vue) — Two cleanup bugs
- `addCleanup(window.removeEventListener(...))` passed `undefined` (return value of `removeEventListener`) instead of a function to `addCleanup`.
- The anonymous arrow function passed to `removeEventListener` was a different reference than the one added — so the listener was never actually removed.

**Fix:** Stored the handler as `const resizeHandler = ...`, used it for both `addEventListener` and `removeEventListener`, and wrapped the cleanup in an arrow function.

## Session 2026-03-03 — Missing `emits` declarations in tool mixins

### 7. [tool.js](../src/mixins/tool.js) — No `emits` declared (Vue 3 warning on every tool interaction)

`tool.js` emitted four events with no `emits:` key, causing Vue 3 to warn
`"Component emitted event X but it is not declared in the emits option"` on every tool drag, select, and delete:

| Event | Location |
|---|---|
| `'object-selected'` | `init_tool()` mousedown handler |
| `'scroll-lock'` | `init_tool()` mouseup + `start_drag()` |
| `'change-settings'` | `set_state()` |
| `'remove-tool'` | `remove_tool()` |

`LineTool.vue` and `RangeTool.vue` also emit `'drawing-mode-off'` (inside `init()`) with no declaration anywhere.

**Fix:** Added `emits: ['drawing-mode-off', 'object-selected', 'scroll-lock', 'change-settings', 'remove-tool']` to `tool.js`. Vue 3 merges mixin `emits` into the component, so this covers all tool overlay components.

### 8. [RangeTool.vue:65](../src/components/overlays/RangeTool.vue) — `this._$emit` typo

`_$emit` does not exist on a Vue instance — was a leftover `_` prefix from an earlier refactor (matching the old `overlay.js` `_$emit` pattern that was already removed in Session 1).

**Fix:** Changed `this._$emit(...)` → `this.$emit(...)`.

## Session 2026-03-03 — Vue3 Event Bus for Tool/Overlay Event Routing

### Root Cause

After removing the Vue2 `$emit` proxy hack in `overlay.js`, tool and pin events (`change-settings`, `scroll-lock`, `object-selected`, `drawing-mode-off`, `remove-tool`) were silently dropped. Grid.vue's `v-on="layer_events"` only handles lifecycle events and `custom-event`, not tool-level events directly. This caused `this.p1`/`this.p2` to always be `undefined` in LineTool/RangeTool, making `draw()` bail immediately — blank canvas for tools.

### Fix: mitt event bus passed as prop

`mitt` v3.0.1 was already a dependency. A per-grid bus is created in Grid.vue, passed to every overlay via `common_props()`, and all tool/pin events route through it back to the existing `custom-event` → DataCube chain.

**Event flow:**
```
pin.js: this.comp.bus_emit('change-settings', { p1: [...] })
  → overlay.js bus_emit(): augment args with [data, grid_id, id, $uuid]
    → layerBus.emit('change-settings', augmented_args)
      → Grid.vue wildcard listener: emit('custom-event', { event, args })
        → Section → Chart → TradingVue → DataCube.on_custom_event()  ✓
```

### 9. [Grid.vue](../src/components/Grid.vue) — Create and subscribe to mitt bus

Added `import mitt from 'mitt'`. Created `const layerBus = mitt()` with a wildcard listener that wraps all bus emissions as `custom-event` for the existing parent chain. Added `bus: layerBus` to `common_props()`. Added `layerBus.all.clear()` cleanup on unmount.

### 10. [overlay.js](../src/mixins/overlay.js) — Add `bus` prop and `bus_emit()` method

Added `'bus'` to the props array. Added `bus_emit(event, ...args)` that replicates the old `$emit` proxy augmentation logic (appends `grid_id`, `id`, `$uuid` to appropriate events; sets `overlay`/`uuid` on `new-interface`) then calls `this.$props.bus.emit(event, args)`. Simplified `custom_event()` to delegate to `bus_emit()`.

### 11. [tool.js](../src/mixins/tool.js) — Replace `this.$emit` with `this.bus_emit`

5 replacements: `object-selected` (mousedown), `scroll-lock` (mouseup + `start_drag()`), `change-settings` (`set_state()`), `remove-tool` (`remove_tool()`).

### 12. [pin.js](../src/components/primitives/pin.js) — Replace `this.comp.$emit` with `this.comp.bus_emit`

7 replacements: constructor `scroll-lock`, `update()` change-settings, `update_from()` change-settings, `mousedown()` scroll-lock × 2 + object-selected, `mouseup()` scroll-lock.

### 13. [LineTool.vue](../src/components/overlays/LineTool.vue) — Route `drawing-mode-off` through bus

`this.$emit('drawing-mode-off')` → `this.bus_emit('drawing-mode-off')` in `init()` settled handler.

### 14. [RangeTool.vue](../src/components/overlays/RangeTool.vue) — Route `drawing-mode-off` and fix pre-existing bug

`this.$emit('drawing-mode-off')` → `this.bus_emit('drawing-mode-off')`. Also fixed a pre-existing bug: `this.$emit('custom-event', { event: 'object-selected', args: [] })` bypassed arg augmentation — changed to `this.bus_emit('object-selected')`.

### 15. [datacube.js](../src/helpers/datacube.js) — `del()` left null slots in `onchart`/`offchart` arrays

**Bug:** `del()` replaced the Vue 2 `this.tv.$delete(obj.p, i)` call (which splices the element out) with `obj.p[i] = null` to preserve Vue 3 reactivity. This left `null` slots in the arrays. `update_ids()` in `dc_core.js` then iterated those arrays without null guards, throwing `TypeError: Cannot read properties of null (reading 'type')` inside `system_tool('Remove')` → `del()` → `update_ids()`. Vue 3 caught this and surfaced it as `"Unhandled error during execution of component event handler at <Chart>"` (pointing to `Chart.vue:328`, the `emit('custom-event', d)` call).

**Fix:** In `del()`, when `i` is a number (whole overlay removal), use `obj.p.splice(i, 1)` instead of null-assignment. When `i` is a string (field key on an object), `null` assignment is still correct. `splice` + `indexOf` is safe in a loop because `indexOf` searches by reference on each iteration, finding the correct current position after any earlier splices.

## Key Lesson
In Vue 3 `<script setup>`, component internals are private by default. Any value or method a parent needs to access via a template ref (`ref.value.x`) must be explicitly listed in `defineExpose()`.
